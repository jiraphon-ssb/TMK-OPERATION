# Blueprint: รายงานเซลล์ + CRM "ลูกค้าใหม่ต่อยอด" + หน้า Import/Export

> วิเคราะห์จากไฟล์จริง `JK Sale Report 2025 - ปาย.xlsx` · เป้าหมายหลัก: **ลูกค้าใหม่ต้องเอาไปต่อยอดได้**

---

## 1) สิ่งที่เจอในไฟล์ (วิเคราะห์แล้ว)

ไฟล์ = **1 เซลล์ ("ปาย") 1 ไฟล์** · 2 ชีต:

### ชีต A: `Database-sale` — log ขายรายออเดอร์ (2,818 แถว · ฿1.7M · 6,400 ชิ้น · ก.ย.2025→มิ.ย.2026)
`วันที่ · ประเภทงาน(DFT/ค้าปลีก) · ชำระ(โอน/COD) · ลูกค้า(ใหม่/เก่า) · จำนวนเงิน · จำนวนตัว · เลขออเดอร์(SIxxxx) · หมายเหตุ`
- **order-level — ไม่มี ลาย/สี/ไซซ์** → ป้อน analytics ระดับออเดอร์ได้ (ยอด/เซลล์/ชำระ/ลูกค้าใหม่-เก่า/เทรนด์) แต่ไม่เข้า lay/variant
- หมายเหตุมี info ดี (แบ่งชำระ/สกรีน/คืนเงิน) — เก็บไว้ใน attrs

### ชีต B: `Sale Report 2025` — KPI รายวัน + **FUNNEL** (298 วัน)
`ยอดขายรายวัน/สะสม · ออเดอร์ · ตัว · `**`คนทัก FB(ใหม่/เก่า) · คนทัก Line(ใหม่/เก่า) · รวมคนทัก · %ปิดการขาย`**` · Basket Size · แยกค้าปลีก/DFT/โอน/COD`
- **🔑 ของใหม่ที่ระบบเราไม่มีเลย:** funnel การตลาด — คนทัก(leads) → ปิดการขาย(conversion) แยกช่อง FB/Line + ใหม่/เก่า

## 2) ⚠️ ปัญหาชี้ขาด: นับซ้ำ (ต้องตัดสินใจ)

- เซลล์ "ปาย" = `paiy.pig1` **มีในข้อมูลมาร์เก็ตเพลสแล้ว 190 ออเดอร์** (Shipnity-direct พ.ค.–มิ.ย.2026)
- order_no ในไฟล์เซลล์ (SI) **ทับกับ DB เรา 187 รายการ**
- ช่วงเวลาคาบเกี่ยว → **รวมดื้อๆ = ฿1.7M + ฿679k นับซ้ำ** เสีย reconcile

**3 ทางเลือก:**
| ทาง | วิธี | ข้อดี | ข้อเสีย |
|---|---|---|---|
| **A. dedup รวม** | นำเข้าเป็น source='เซลล์' + ลบ/ข้าม order_no ที่ซ้ำ DB | ยอดรวมเดียว ครบ | dedup ต้องแม่น (SI≠SK อาจเป็นออเดอร์เดียวกันคนละเลข) |
| **B. แยกชั้น** | เก็บ source='เซลล์' แต่ dashboard มี toggle "นับเซลล์ไหม" | ปลอดภัย ไม่ปน | ต้องเลือกมุมมอง |
| **C. เฉพาะ funnel** | เอาแค่ชีต B (funnel) ไม่เอาชีต A เข้ายอด | 0 ความเสี่ยงนับซ้ำ + ได้ของใหม่ | ไม่ได้ history ขายของเซลล์ |

> **แนะนำ: เริ่ม C (funnel ปลอดภัย+ใหม่) ก่อน → แล้วทำ A/B (ขายเซลล์) เมื่อ confirm วิธี dedup**

## 3) ระบบที่ควรมี — แบ่งระดับ (จำเป็น / ควรมี / มีก็ดี)

### 🟥 จำเป็น (ทำเลย)
1. **หน้า Import/Export Hub** (catalog sub ใหม่) — ย้าย import มาที่นี่ + ส่งออก + ประวัตินำเข้า + (ใหม่) อัปโหลดไฟล์เซลล์
2. **ระบบ Funnel/Leads** — ตาราง `tmk_sales_funnel` (date·salesperson·leads FB/Line ใหม่-เก่า·orders·sales·close_rate) + import ชีต B + section ใน dashboard: คนทักรวม · %ปิดการขาย · leads ใหม่ตามเวลา · FB vs Line · conversion ต่อเซลล์
   - **ตรง "ลูกค้าใหม่ต่อยอด" โดยตรง:** เห็น lead ใหม่เข้าเท่าไร ปิดได้กี่ % ช่องไหนดี เซลล์ไหนเก่ง

### 🟧 ควรมี (ต่อยอดจริง)
3. **ขายเซลล์เข้าระบบ** (ทาง A/B) — source='เซลล์' → ทุกแท็บนับเซลล์รวม + ทีมขาย leaderboard มีคนจริง + history ย้อนถึง ก.ย.2025 (เพิ่ม timeline)
4. **CRM ลูกค้า + Follow-up** — สร้าง cohort "ลูกค้าใหม่" ต่อเดือน → ติดตาม: กลับมาซื้อ %, ควร win-back, ควร upsell. ผูกกับ RFM tier ที่มีแล้ว → list "ลูกค้าใหม่เดือนนี้ที่ยังไม่ซื้อซ้ำ" ให้เซลล์ตาม
5. **เป้า vs จริง (target)** — เอา Gauge เป้ากลับ (หายตอนเปลี่ยน dashboard) + เทียบเป้าต่อเซลล์/วัน

### 🟨 มีก็ดี (อนาคต)
6. **ฟอร์มเซลล์กรอกในเว็บ** (ไม่ต้องใช้ Excel) — เซลล์กรอกขาย/ลูกค้าใหม่ + เบอร์/ช่องทางติดต่อ → ได้ **contact จริงไว้ follow-up** (Excel ตอนนี้มีแค่นับ ใหม่/เก่า ไม่มีเบอร์)
7. **แจ้งเตือนติดตาม** — ลูกค้าใหม่ที่ไม่ซื้อซ้ำใน N วัน → เตือนเซลล์
8. **วิเคราะห์หมายเหตุ** (แบ่งชำระ/คืนเงิน/สกรีน) → flag operational

## 4) Data mapping (ชีต A → ระบบเดิม)
| Excel | → field เรา |
|---|---|
| วันที่ | order_date |
| ประเภทงาน (DFT/ค้าปลีก) | job_type |
| ประเภทการชำระ (โอน/COD) | payment_type |
| ลูกค้า (ใหม่/เก่า) | customer_type |
| จำนวนเงิน | sales |
| จำนวนตัว | qty |
| เลขออเดอร์ (SI) | order_no (dedup key) |
| หมายเหตุ | attrs.note |
| (ชื่อไฟล์ "ปาย") | salesperson · channel='เซลล์' · source='เซลล์' |
ไม่มี: ลาย/สี/ไซซ์ (order-level), จังหวัด, ชื่อลูกค้า → SKU/geo ของเซลล์จะว่าง ( honest)

## 5) Data model ใหม่
- `tmk_sales_funnel` (ใหม่ · migration): id(date:salesperson), date, salesperson, leads_fb_new/old, leads_line_new/old, orders, sales, qty, close_rate, attrs. UI-gate.
- ขายเซลล์ → reuse `tmk_mp_orders` (source='เซลล์') — ไม่ต้อง migration

## 6) ลำดับงาน (เริ่มเลย)
- **C1 หน้า Import/Export Hub** (ย้าย import + ส่งออก + ช่องอัปโหลดไฟล์เซลล์)
- **C2 ตาราง funnel + pipeline parse ชีต B + import**
- **C3 section Funnel/Leads ใน dashboard** (KPI คนทัก/ปิดการขาย/เทรนด์/ต่อเซลล์)
- **C4 (รอ confirm overlap) ขายเซลล์เข้า source='เซลล์' + dedup**
- **C5 CRM follow-up** (cohort ลูกค้าใหม่ + list ตาม) + เป้ากลับมา
- **C6 (อนาคต) ฟอร์มกรอกในเว็บ + contact + แจ้งเตือน**
