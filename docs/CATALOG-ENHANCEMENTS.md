# แคตตาล็อก — ข้อเสนอเพิ่มเติม (อิงรายงานยอดขายรวม Juntakarn)

> สังเคราะห์จาก **Handoff Spec รายงานยอดขายรวม** (Shopee/TikTok/Shipnity + แคตตาล็อก 86 ลาย) เทียบกับหน้าแคตตาล็อกปัจจุบันในแอป
> สถานะ: **เฟสเสนอ — ยังไม่ลงมือ** (รอเลือกว่าจะทำตัวไหนก่อน)

---

## 1. บริบทธุรกิจจริง (สิ่งที่รายงานบอก)

- แบรนด์เสื้อโปโลลายไทย — **คิดเป็น "ลาย" (design) ไม่ใช่ "สินค้าเดี่ยว"**: 1 ลายมีหลาย LOT (JSK01/02/03 = สิริกานต์), 1 ลายขายหลายสี×ไซซ์
- **ขาย 7 ช่อง**: Shopee · Lazada · Facebook · LINE · โทร · หน้าร้าน POS · TikTok
- **Insight สำคัญสุด: แต่ละลายขายคนละช่อง** — สีดำ-อปท/กระทรวง = Shopee เท่านั้น · จันทร์/กนก/ลอยกระทง = ช่องตรงเท่านั้น · สิริกานต์ = ดาวเด่นทุกช่อง · TikTok นิยมไม่ซ้ำ Shopee → **วางแผนสต็อก/โปรต้องแยกช่อง**
- ไซซ์ขายดี: **M > L > S > XL > XS ≈ 2XL > 3XL > 4XL > 5XL**
- ต้องการมิติ **"ประเภทงาน" (ค้าปลีก / ส่ง / OEM-หน่วยงาน)** — ยังไม่มี
- โครง 3 ชั้นที่เขาใช้: **ลาย → สี/ไซซ์ (SKU ย่อย) → สต็อก (รวม/จอง/เหลือ)**

---

## 2. ✅ สิ่งที่แคตตาล็อกมีแล้ว (ของดี ไม่ต้องทำซ้ำ)

- **ProductsView**: ตารางสินค้า + ค้นหา/เรียง(5)/ฟิลเตอร์สถานะ + CSV + ป้ายบาร์โค้ด
- **StockView**: matrix ไซซ์×สินค้า + drill-down สี×ไซซ์ต่อสินค้า + KPI + **การจอง (ATP)** + แจ้งเตือนใกล้หมด + ประวัติเคลื่อนไหว
- **SalesReportView (เด่นมาก)**: ขายดี · **ไซซ์ขายดี · สีขายดี** · กำไรตามหมวด · **ABC/Pareto 80/20** · **sell-through** · **aging/turnover** · มูลค่าคลังตามเวลา · รายวัน/รายเดือน · CSV 10 ตาราง
- โครงสินค้า: **lots (สี×ไซซ์ grid + ต้นทุน + lotNo + วันที่)** · variants · reservedByVariant · available(ATP) · oldestLotDate
- **`tmk_sales` เก็บ `channel` ต่อรายการขายแล้ว** (จาก `order.channel`) — สำคัญมากต่อข้อ 4.1

โครง 3 ชั้นของเขา (ลาย→สี/ไซซ์→สต็อก) **แอปทำชั้น 2-3 ได้แล้ว** (lots = สี×ไซซ์×สต็อก) ขาดแค่ชั้น 1 "ลาย" ที่ข้ามหลาย LOT

## 3. 🎯 ช่องว่าง (map กับความต้องการธุรกิจ)

| Gap | ธุรกิจต้องการเพราะ | ข้อมูลพร้อมไหม |
|---|---|---|
| **ยอดขายแยกช่องทาง ต่อสินค้า/ลาย** | insight #1 — แต่ละลายขายคนละช่อง | ✅ มีแล้ว (`tmk_sales.channel`) แค่ยังไม่ breakdown |
| **"ลาย" (design grouping) ข้าม LOT** | คิดเป็นลาย, รวม LOT เป็นยอดเดียว | ⚠️ ต้องเพิ่มฟิลด์ `design`/`pattern` หรือ derive |
| **ประเภทงาน (ปลีก/ส่ง/OEM)** | แยกตลาด, หน่วยงานสั่งล็อตใหญ่ | ❌ ต้องเพิ่มฟิลด์ที่ออเดอร์ |
| **เติมสต็อกอัจฉริยะ (velocity/วันของหมด)** | วางแผนผลิตทันก่อนหมด | ✅ มี sold + onHand แค่คำนวณ velocity |
| **นำเข้ายอดจาก export มาร์เก็ตเพลส** | รายงานทั้งระบบมาจากตรงนี้ | ⚠️ ฟีเจอร์ใหม่ (ผูกกับ AI roadmap) |
| **เทียบงวด (เดือนนี้ vs เดือนก่อน)** | ดูโต/ตกต่อสินค้า | ✅ มี order_month |
| **การ์ดสรุปสินค้า (scorecard)** | ดูสุขภาพสินค้าตัวเดียวครบจอ | ✅ มีข้อมูลครบ แค่จัดรวม |
| **ลูกค้าซื้ออะไร (B2B/หน่วยงาน)** | ลูกค้าหน่วยงานซื้อซ้ำลายเดิม | ✅ orders มี customer + items |

## 4. 🚀 ข้อเสนอ (จัดลำดับตามคุ้มค่า ÷ แรง)

### 🟢 กลุ่ม A — คุ้มสุด + ข้อมูลพร้อม (ทำได้เลย ไม่ต้องเพิ่ม schema)

**A1. ⭐ ยอดขายแยกช่องทาง (Channel breakdown)** — *ตรง insight #1*
- เพิ่มใน SalesReportView: ตาราง **"ลาย/สินค้าขายดี × ช่องทาง"** (แถว=สินค้า, คอลัมน์=ช่องทาง facebook/tiktok/shopee/lazada/line/crm + รวม) เหมือน Top-15 ในรายงาน
- + การ์ด **"ยอดขายตามช่องทาง"** (bar) ในแคตตาล็อก (ตอนนี้มีแค่ใน Dashboard)
- + ในการ์ดสินค้า/scorecard: mini-breakdown "ช่องไหนขายดี"
- **ข้อมูล:** `tmk_sales` มี `channel` + `product_id` + `qty` + `amount` แล้ว → SUMIFS ได้ทันที
- effort: **S–M** · impact: **สูงสุด**

**A2. เติมสต็อกอัจฉริยะ (Reorder / days-of-stock)**
- ในแจ้งเตือน StockView + scorecard: คำนวณ **อัตราขาย/วัน** (จาก tmk_sales ช่วง 30–90 วัน) → **"เหลือพออีก N วัน"** + **แนะนำสั่งผลิต** ให้ถึงระดับ X วัน (เก่งกว่า reorder_point คงที่)
- เน้นลายที่ velocity สูง × ใกล้หมด = "ต้องสั่งด่วน"
- effort: **M** · impact: **สูง** (กันของขาดช่วงพีค)

**A3. การ์ดสรุปสินค้า (Product Scorecard)**
- กดสินค้า → panel/หน้าเดียวรวม: ยอดขาย·ต้นทุน·กำไร·มาร์จิ้น · turnover · aging · sell-through · **ช่องทางที่ขาย** · สี/ไซซ์ขายดีของตัวนี้ · กราฟ trend · ล็อต+สต็อกคงเหลือ
- รวมของที่กระจายใน SalesReport มาไว้ "ต่อสินค้า"
- effort: **M** · impact: **สูง**

**A4. เทียบงวด (เดือนนี้ vs เดือนก่อน)**
- SalesReportView เพิ่มลูกศร ↑↓ % เทียบงวดก่อน (ต่อสินค้า/หมวด/ช่องทาง) — รู้ว่าตัวไหนโต/ตก
- effort: **S** · impact: **กลาง**

### 🟡 กลุ่ม B — เพิ่ม dimension (ต้องเติม schema นิดหน่อย)

**B1. "ลาย/Design" grouping ข้าม LOT** — *โครงชั้น 1 ที่ขาด*
- เพิ่มฟิลด์ `design`/`pattern` ที่สินค้า (เช่น JSK01/02/03 → design="สิริกานต์") — ใส่เองหรือ derive จากชื่อ
- เพิ่มมุมมอง **"ตามลาย"** ใน Products/Stock/Sales: รวม LOT เป็นยอดเดียว + **สต็อกคงเหลือต่อลาย** (roadmap §15.1 ของรายงาน)
- toggle "ดูตามลาย / ตาม LOT"
- effort: **M** · impact: **สูง** (ตรงวิธีคิดของธุรกิจ)

**B2. ประเภทงาน (ค้าปลีก / ส่ง / OEM-หน่วยงาน)**
- เพิ่มฟิลด์ `job_type` ที่ออเดอร์ (เลือกตอนสร้าง) → breakdown ยอด/กำไรตามประเภทงาน
- ระหว่างรอ: ใช้ qty_band (>10 ตัว = น่าจะส่ง/หน่วยงาน) เป็น proxy เหมือนรายงาน
- effort: **M** · impact: **กลาง–สูง** (แยกตลาดหน่วยงานที่เป็นรายได้ก้อนใหญ่)

**B3. ลูกค้าซื้ออะไร (Customer purchase detail)**
- หน้า/แผงลูกค้า: รายการที่เคยซื้อ (ลาย/สี/ไซซ์/ช่องทาง/ครั้งล่าสุด) + แนะนำ "ลูกค้าหน่วยงานนี้ซื้อซ้ำลายเดิม"
- effort: **M** · impact: **กลาง** (ดูแลลูกค้า B2B)

### 🔵 กลุ่ม C — ฟีเจอร์ใหญ่ (เชื่อม pipeline เข้าแอป)

**C1. ⭐ นำเข้ายอดขายจาก export มาร์เก็ตเพลส** — *เอา pipeline ทั้งหมดเข้าแอป*
- ลากวาง `shopee.xlsx / tiktok.xlsx / Shipnity.xlsx` → แอป parse + จับคู่ catalog + กันนับซ้ำ (D3: Shopee อยู่ใน Shipnity 100%) → เติม `tmk_sales` + ออเดอร์
- ผูกกับ **AI roadmap** (parse ด้วย Claude/JS) + เข้ากับ MEMORY (เว็บเป็นช่องกรอกข้อมูล)
- ได้ทุก analytics ในข้อ A/B จากข้อมูลจริงทันที (ไม่ต้องกรอกมือ)
- effort: **L** · impact: **สูงมาก** (แต่ซับซ้อน — SKU matching 5 วิธี, dedup, channel mapping)

**C2. แยก LOT ในรายงาน + ต้นทุน/กำไรต่อ LOT**
- ยอดขาย/กำไรแยกตาม LOT (รายงาน §15.7) — รู้ว่าล็อตไหนคุ้ม
- ต้องผูก lot id เข้า sale record (ตอนนี้เก็บ color/size แต่ไม่เก็บ lot)
- effort: **M–L** · impact: **กลาง**

## 5. 🗺️ ลำดับที่แนะนำ

**เฟส 1 (คุ้มสุด-เร็วสุด, ข้อมูลพร้อม):** A1 ยอดขายแยกช่องทาง → A3 scorecard → A2 เติมสต็อกอัจฉริยะ → A4 เทียบงวด
**เฟส 2 (ตรงวิธีคิดธุรกิจ):** B1 ลาย/design grouping → B2 ประเภทงาน → B3 ลูกค้าซื้ออะไร
**เฟส 3 (ฟีเจอร์ใหญ่):** C1 นำเข้า export มาร์เก็ตเพลส (เอา pipeline เข้าแอป) → C2 แยก LOT

> **เริ่มที่ A1** ดีสุด — impact สูงสุด (ตรง insight #1), **ข้อมูลมีครบใน `tmk_sales` แล้ว**, ไม่ต้องแตะ schema/migration, ทำเสร็จเห็นผลทันที

## 6. หมายเหตุเชื่อมกับของเดิม
- A1/A2/A3/A4 ใช้ `tmk_sales` (มี channel/product_id/qty/amount/cost/lines) + computeMonth — ไม่ต้อง migration
- B1/B2 ต้อง `alter table` เพิ่มคอลัมน์ (design / job_type) — รันใน SQL Editor (มี fallback ถ้ายังไม่รัน ตาม pattern เดิม)
- C1 เข้ากับ AI roadmap §"ถ่าย/วางสรุปยอด → RecordSalesModal" และ "นำเข้าออเดอร์จาก export"
- ทุกข้อ: read-only analytics + เขียนผ่าน modal เดิม (ไม่ชน MEMORY: เว็บเป็นช่องกรอกข้อมูลทางเดียว)
