## เป้าหมายของ handoff

ให้ Claude ไปต่อจากจุดนี้ได้ทันที โดยไม่ต้องเริ่มสำรวจใหม่ทั้งหมด:

- เข้าใจภาพรวมระบบ TMK Operation
- รู้ว่าส่วนไหนใช้งานจริงแล้ว ส่วนไหนยังเป็นงานค้าง
- รู้ลำดับงานที่ควรทำก่อน
- รู้จุดเสี่ยงเรื่องข้อมูล/UX/เทคนิค

## ภาพรวมระบบ

ระบบเป็น React/Vite app สำหรับบริหารงานแบรนด์แฟชั่น TMK มี Supabase เป็น backend หลัก

Tech stack:

- React 19
- Vite 8
- Supabase
- Chart.js
- xlsx
- Custom CSS ไม่มี Tailwind/TypeScript

ไฟล์สำคัญ:

- `src/App.jsx` shell, navigation, auth, modal host
- `src/dataContext.jsx` โหลด Supabase หลักของระบบ operation เดิม
- `src/views-1.jsx` Home + Sales เดิม
- `src/views-2.jsx` Planner + Sale hub + Settings/Health
- `src/saleDashboard.jsx` รายงานขายใหม่จาก import
- `src/saleCrm.jsx` CRM ลูกค้า
- `src/saleCatalog.jsx` แคตตาล็อกเสื้อ
- `src/saleImportHub.jsx` Import/Export + reconciliation
- `src/lib/saleData.js` data cache/fetch layer ของ Sale
- `src/lib/saleAgg.js` aggregation รายงานขาย
- `src/lib/saleRecon.js` parse/reconcile ไฟล์เซลล์

## Priority ที่แนะนำ

### P0 — Data Readiness ก่อน

เป้าหมาย: ทำให้รายงานเชื่อถือได้ก่อนนำไปตัดสินใจ

งาน:

1. เคลียร์ Data Health
   - ผูกลาย `จันทกานต์`, `ดารารัตน์`, `กระทงจ๋า`
   - ตรวจคำที่ระบบตีเป็นสีผิด เช่น `คอกลม`, `คอวี`
   - ผูก SKU/ชื่อจาก TikTok/Shopee/Shipnity ที่ยังจับคู่ไม่ได้

2. ตั้ง alias เซลล์
   - `jirarattukta`
   - `paiy.pig1`
   - `Oum Juntakarn`
   - `khing.thaipolo1`

3. เติม catalog ให้ครบ
   - ราคา 0
   - ต้นทุน
   - สถานะสินค้า
   - หมวด/ลาย/สี/ไซซ์

Acceptance criteria:

- Data Health จับคู่ลายได้อย่างน้อย 98%
- มีรหัสลายอย่างน้อย 98%
- รายงานทีมขายไม่แตกชื่อเซลล์หลัก
- Catalog ไม่มีสินค้าหลักราคา 0 โดยไม่ตั้งใจ

### P1 — CRM Follow-up Workflow

เป้าหมาย: ทำให้ลูกค้า 658 รายที่ติดต่อได้กลายเป็นงานขายจริง

งาน:

1. ทำหน้า/โหมด `คิวตามต่อ` ให้เป็น action list
2. เพิ่มสถานะ follow-up:
   - ยังไม่ตาม
   - โทรแล้ว
   - ทักแล้ว
   - สนใจ
   - ปิดการขายได้
   - ไม่สนใจ
   - นัดตามอีกครั้ง
3. เพิ่ม note/next follow-up date
4. ทำ filter `ลูกค้าใหม่มีเบอร์แต่ยังไม่ซื้อซ้ำ`
5. ผูกกับเซลล์เจ้าของลูกค้า

Acceptance criteria:

- เซลล์เปิด CRM แล้วรู้ทันทีว่าต้องตามใครก่อน
- กดจบ action ได้ใน 1-2 คลิก
- ผู้บริหารเห็นจำนวนคิว/ผลตามต่อ/ยอดที่เกิดจาก follow-up

### P2 — Funnel: คนทัก & ปิดการขาย

เป้าหมาย: เติม missing link ระหว่าง marketing lead กับ sales

งาน:

1. ตรวจ migration `tmk_sales_funnel` ว่ารันแล้วหรือยัง
2. ทำ flow กรอกคนทักวันนี้ให้ใช้ง่าย
3. เชื่อมรายงาน `คนทัก & ปิดการขาย`
4. แสดง conversion ต่อเซลล์/ช่องทาง

Acceptance criteria:

- หน้า `คนทัก & ปิดการขาย` ไม่ว่าง
- เห็น leads FB/LINE ใหม่/เก่า
- เห็น close rate ต่อเซลล์
- เห็นแนวโน้ม leads vs orders

### P3 — Clarify UX ระหว่าง `ยอดขาย` กับ `Sale`

ปัญหา:

- มีสองหน้าที่เกี่ยวกับยอดขาย
- ตัวเลขไม่เท่ากันเพราะ source ต่างกัน
- ผู้ใช้มีโอกาสงง

ข้อเสนอ:

- `ยอดขาย`: operation dashboard, เป้า, pace, daily entry, budget

งาน:

1. ปรับชื่อเมนูหรือคำอธิบาย
2. ใส่ source label ให้ชัด
3. ทำ bridge card เช่น จาก `ยอดขาย` ไป `Sale report`
4. ลด duplication ของ KPI ที่ทำให้เข้าใจผิด
